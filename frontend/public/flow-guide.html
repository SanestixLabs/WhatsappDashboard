Node Types
Each node does one job. Connect them together to build your automation.
Send Message
Sends a text message to the contact. Supports variables like {name}. Automatically moves to the next node.
Capture Input
Sends a prompt and WAITS for the contact's reply. Saves the reply as a named variable (e.g. name, order_id).
Condition
Checks a variable value. If condition is met, goes one way; otherwise the other. Used for branching logic.
Delay
Waits a set number of seconds before moving to the next node. Useful for pacing conversations naturally.
Human Handoff
Pauses automation and routes the conversation to a human agent. Optionally sends a message before handing off.
AI Reply
Hands the conversation back to the AI assistant. The flow ends and normal AI responses resume.
End Flow
Ends the automation session. Optionally sends a final message. Always add this as the last node.
2
Variables — How to Use Them
Variables let you capture what the contact says and use it later in messages.
Variable Format
{variable_name}—Use curly braces. No spaces. Lowercase only.
ℹ️
Variable names must match exactly between the Capture Input node (where you define it) and the Send Message node (where you use it). Case-sensitive.
Step-by-step example
Step 1
Add a Capture Input node
Prompt: What is your name?
Variable Name: name
Step 2
Contact replies
Contact sends: Ahmed
System stores: {name} = "Ahmed"
Step 3
Add a Send Message node
Message: Hi {name}, welcome to our store!
Contact receives: Hi Ahmed, welcome to our store!
Variable NameCapture Input FieldUse in MessageExample Output
nameVariable Name: name{name}Hi Ahmed
order_idVariable Name: order_id{order_id}Order #1234
phoneVariable Name: phone{phone}+92 300 123 4567
cityVariable Name: cityDelivering to {city}Delivering to Lahore
⚠️
Common mistakes: Using { name } (spaces inside braces) or {Name} (capital N) will NOT work. Must be exactly {name}.
3
Building a Flow — Step by Step
From empty canvas to working automation in 5 minutes.
1
Create the flow
Click + New Flow, give it a name, choose a trigger type (Keyword, First Message, or Button Reply), and enter the trigger value if needed.
2
Add nodes from the left panel
Click any node type in the left panel to add it to the canvas. Nodes appear in the center — drag them to reposition.
3
Configure each node
Click a node on the canvas to open its config panel on the right. Fill in the message text, variable name, delay, etc. Click Save Node.
4
Connect the nodes
Hover over the bottom dot of a node — cursor changes to a crosshair. Drag from that dot to the top dot of the next node.
5
Save the flow
Click the Save button in the top-right. You'll see "Flow saved!" if successful.
6
Activate the flow
Click Activate. The status dot turns green. The flow is now live.
Visual Flow Example
Trigger
Keyword: "order"
Capture Input
Prompt: "What is your order ID?" | Variable: order_id
Send Message
"Got it! Checking order {order_id} for you..."
End Flow
Final message: "Thank you! Our team will follow up shortly."

Choose how your flow starts when a contact messages you.
TriggerWhen it firesTrigger ValueExample
KeywordContact's message contains the keywordThe keyword to match (e.g. order)Contact sends "check order status" → matches keyword "order"
First MessageFirst time a contact ever messages youNot requiredNew contact sends any message → welcome flow starts
Button ReplyContact taps a WhatsApp buttonThe button payload/text to matchContact taps "Track Order" button → flow starts
✓
Keyword tip: The keyword match is case-insensitive and checks if the message CONTAINS the keyword. "check ORDER status" will match keyword "order".

Tips & Rules
Things to keep in mind for flows that work reliably.
✓
Always end with End Flow node. Without it, the session stays open forever.
✓
Connect every node. Drag from the bottom dot to the top dot of the next node.
✓
Save Node first, then Save Flow. Click "Save Node" after editing, then "Save" in the top bar.
⚠️
Variable names: no spaces, no capitals. Use order_id not Order ID.
⚠️
Capture Input waits for ONE reply. The contact's next message is captured as the variable.
ℹ️
Sending keyword again restarts the flow. Current session ends and flow restarts from beginning.
ℹ️
Flow messages appear in chat. All bot messages show up in the Sanestix chat window in real-time.
